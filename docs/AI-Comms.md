### Message Infrastructure

**Pub/Sub Pattern:**
```
Broker.publish(channel, message)
  → Redis PUBLISH {channel} {message_json}
  → Listeners dispatch to all subscribed handlers

Broker.subscribe(channel, handler)
  → Start background listener task
  → Register handler callback
  → Handler called async for each message
```

**Queue Pattern:**
```
Broker.enqueue_task(agent_id, task)
  → RPUSH voidcat:inbox:{agent_id} {task_json}
  → Task added to agent's inbox

Broker.dequeue_task(agent_id)
  → LPOP voidcat:inbox:{agent_id}
  → Returns oldest task (FIFO)
```

**Caching Pattern:**
```
Broker.set_cache(key, value, ttl)
  → SET key {value_json} [EX ttl]
  → JSON serialized value

Broker.get_cache(key)
  → GET key
  → Returns deserialized value or None
```

### Redis Data Structures
