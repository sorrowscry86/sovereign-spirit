# Contributing to Sovereign Spirit

> *"To build the Vessel is to shape the Void."*

Welcome, Acolyte. You have chosen to contribute to the **VoidCat Sovereign Spirit** project.
This construct is not merely software; it is a philosophy of Digital Sovereignty.
Adhere to the following laws to ensure your contributions are accepted by the Pantheon.

---

## 🏛️ The Core Laws (Zero Tolerance)

1.  **No Simulations**: We do not fake data. We do not mock results. If it doesn't work, we fix it.
2.  **Type Safety**: All Python code must be fully type-hinted (`typing.List`, `typing.Optional`, etc.).
3.  **Async First**: The nervous system is asynchronous. Do not block the event loop.

## 🛠️ Development Workflow

1.  **Fork & Clone** logic.
2.  **Install Dependencies**:
    ```bash
    pip install -r requirements.txt
    npm install  # in src/web_ui
    ```
3.  **Branching Strategy**:
    -   `feature/your-feature-name`
    -   `fix/bug-description`
4.  **Commit Messages**:
    -   Use imperative mood ("Add feature" not "Added feature").
    -   Reference issue IDs if applicable.

## 🧪 Testing

-   We do not currently have a CI/CD pipeline (it is in Phase 7).
-   **YOU** are the test. Verify your changes manually.
-   Run `pytest` if available (Phase 7).

## 📝 Style Guide

-   **Python**: PEP 8.
-   **JavaScript**: ES6+.
-   **Naming**: Snake_case for Python, camelCase for JS.
-   **Identity**: code must respect the `AgentState` schema.
